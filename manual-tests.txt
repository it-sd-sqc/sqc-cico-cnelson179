Module 4: Ticket 402 - Constrain developer card number input length

Preconditions:
- Application is running.
- You are on the screen that contains the "Developer Card Number" JTextField.
- CARD_NUMBER_MAX = <PUT_MAX_LENGTH_HERE>

Test Steps:
1) Click inside the Developer Card Number text field.
2) Type digits until you reach CARD_NUMBER_MAX digits.
3) Try typing 3 more digits after reaching the limit.
4) Copy a long string of digits longer than CARD_NUMBER_MAX (ex: 999999999999999) and paste it into the field.
5) Select all text in the field and paste a string longer than CARD_NUMBER_MAX again.
6) Try typing non-digit characters (letters/symbols) into the field.

Expected Results:
- The field accepts digits up to CARD_NUMBER_MAX.
- After CARD_NUMBER_MAX digits, additional digits are not inserted.
- Pasting longer-than-max input results in the field containing no more than CARD_NUMBER_MAX digits (extra characters are rejected).
- Non-digit characters are rejected (nothing inserts).
